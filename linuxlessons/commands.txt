Linux commands
ctrl+alt+T=команда для открытия терминала(Spotlight)

~ - символ домашней директории
/ - символ корневой директории
. . - обозначают предыдущую директорию
. - обозначает текущую директорию ю




1. ls - команда для вывода всех директорий (папка) и файлов в текущей директории в терминале

2. Cd(change directory) - команда для изменения (переходов) одной директории на другую
    cd + название директории

3. mkdir(make directory) - создает указанную директорию в рабочей директории в терминале

4. Touch - создает указанный файл

5. rm - удаление файла

6. rmdir - удаление директории

7. Pwd (present working directory) - выводит путь до текущей директории

8. Rm -rf - команда принудительного удаления

9. ls -a - выводит список как скрытых, так и обычных файлов и директории

10. nano <path_to_file> - откроется файл для редактирования в графическом интерфейсе
* если нет файла, то nano попытается создать новый файл

11. cat <file_name> - выводит содержимое файла в терминал

12. echo - команда для быстрой записи текстовой информации в какой-либо файл.
    echo текст_для_записи > название_файла

13. mv - команда для переименовывания файла либо директории, также для перемещения файлов или директорий.
   Переименовывание: mv john john
                     mv <старое название> <новое название>
   Перемещение: mv john ~/Desktop/
                mv <file/directory> <directory>/

GIT - распределенная система для размещения git-репозиториев в совместимой разработке проэктов.
Репозиторий - хранилище вашего кода и истории его изменений

Команды Git:
1. git init - она создает новый локальный гит репозиторий у вас на пк, в той директории в которой была прописана. (Команда вводится один раз в начале)

2. Git add - команда для добавления файлов в систему отслеживания гита, после это команды изменения начинает отслеживается гитом. 

3. Git commit -m '<comment>' - как только достигаем определенного момента в разработке, то тогда мы сохраняем и комментируем изменения в локальной репозитории

4. Git remote add - это команда для того, чтобы связать наш локальный репозиторий с удаленным репозиторием, то есть добавляет подключение к репозиторий на гитхабе
   Git remote add <name: origin> <ссылка_на_peno:ssh>

5. Git push - команда для отправки всех ваших изменений на локальном репозитории в удаленном репозитории
   Git push origin main
